# Qt. Лабораторная работа 3

**Language learning project**

_В ходе данной лабораторной работы вам предлагается написать приложение для изучения иностранного языка. В качестве вдохновения будем отталкиваться от классики жанра -_ [_DuoLingo_](https://www.duolingo.com/)_, однако любая креативность и новаторство с вашей стороны приветствуются (и даже будут поощряться)._

**1\. Требования**

- _Layout._ Размещение виджетов должно быть организовано с помощью компоновщика. Следовательно, ожидается, что при изменении размера окна все элементы на нем будут также пропорционально изменяться.
- _Dynamic widgets._ Ожидается, что часть с общим меню и часть непосредственно с упражнениями будут располагаться на одном окне. Т.к. сами упражнения будут иметь различную структуру (i.e. различные виджеты), работу с динамически изменяющимися компонентами компоновщика можно сделать, например, через [QStackedWidget](https://doc.qt.io/qt-5/qstackedwidget.html) или через явное удаление/добавление компонент.
- _Menu._ В приложении должно присутствовать menuBar как минимум с возможностью изменить уровень сложности. Непосредственно выбор уровня реализовывать через отдельное диалоговое окно _(диалоговые окна будут рассмотрены чуть позже на паре)_.
- _PushButton + TextEdit._
  - Кнопка "Translation", по нажатию которой на части окна справа появляется упражнение по переводу текста. Ответ пользователь вводит с клавиатуры.
  - По нажатию кнопки "Submit" должна происходить оценка введенного перевода + переход к следующему предложению.
- _PushButton + RadioBox._
  - Кнопка "Grammar", при нажатии которой должно отображаться некоторое упражнение с несколькими вариантами ответа и одним единственным верным (например, подстановка глагола в предложение в подходящем времени).
  - По нажатию кнопки "Submit" должна происходить оценка выбранного варианта + переход к следующему предложению.
- _Progress bar._ Каждый новый запуск упражнения должен генерировать некоторый набор из N заданий, которые будут последовательно показаны пользователю. Для визуализации количества уже выполненных заданий добавьте progress bar.
- _Grading system._ За корректное выполнение всего упражнения пользователю должны начисляться баллы. Данные баллы должны отображаться где-нибудь в вашем приложении. Также введите систему для контроля количества неверных попыток. В случае, если пользователь ввел неверный ответ M раз, завершите все упражнение с соответствующим сообщением. Баллы должны начисляться только в случае корректного выполнения всех N заданий.
- _Timer._ При нажатии кнопки начала упражнения должен запускаться таймер. По истечении определенного времени упражнение должно завершаться и пользователю должно выдаваться сообщение о том, что время, отведенное на выполнение задания, истекло.
- _Help._ При нажатии клавиши "H" должно появиться информационное окно с небольшой подсказкой по текущему упражнению. Например, правило спряжения глаголов.
- _Design._ Ожидается, что ваше приложение будет выглядеть презентабельно и будет иметь понятный для пользователя функционал.

**2\. Система оценивания**

При выполнении всех пунктов выше можно получить **10 баллов**. Однако, вы можете также получить до **2 дополнительных баллов** при использовании некоторых advanced features, которые не покрывались лекциями и требуют самостоятельной углубленной работы.

Гарантированные дополнительные баллы:

● Использование аудио в вашем приложении

● Продвинутый алгоритм для сравнения совпадения двух строк (оригинала и перевода). Явное побуквенное сравнение не всегда является оптимальным подходом, например, для языков как французский, где присутствует много символов, которые не встречаются на обычной US раскладке клавиатуры. Также некоторые языки допускают перестановку некоторых слов без потери общего смысла предложения.
