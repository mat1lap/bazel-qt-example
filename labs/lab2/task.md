## Лабораторная работа № 2: Рейкастер

Дедлайн: см.

Попробуем написать **2D-рейкастер**. Это штука, которая симулирует освещение на плоскости. Чтобы понять, как это выглядит и какой результат (приблизительно) ожидается, ознакомьтесь с [этой статьей](https://ncase.me/sight-and-light/).

**В данной лабораторной запрещено использование QPolygon, а также прочих Qt-шных средств для вычислений геометрии, например, QLineF. Другими словами, в Ray, Polygon и Controller использование Qt должно быть сведено к структуре QPoint.**

**1: Многоугольники. Везде многоугольники! (7.5 баллов)**

Создайте класс Ray (луч). Публичный интерфейс:

- Ray(const QPoint& begin, const QPoint& end, double angle)
- Геттеры и сеттеры для параметров.
- Ray Rotate(double angle) const - создает луч, который получается в результате поворота текущего луча на angle радиан.

_Примечание:_ Здесь и далее вместо QPoint возможно использование QPointF.

Создайте класс Polygon. Публичный интерфейс:

- Polygon(const std::vector&lt;QPoint&gt;& vertices)
- Геттер для вершин.
- void AddVertex(const QPoint& vertex)
- void UpdateLastVertex(const QPoint& new_vertex) - изменяет последнюю добавленную вершину на new_vertex.
- std::optional&lt;QPoint&gt; IntersectRay(const Ray& ray) - возвращает точку первого (т. е. ближайшего к ray.getBegin()) пересечения луча с многоугольником, или std::nullopt, если луч не пересекает многоугольник.

Создайте класс Controller. Публичный интерфейс:

- const std::vector&lt;Polygon&gt;& GetPolygons()
- void AddPolygon(const Polygon&)
- void AddVertexToLastPolygon(const QPoint& new_vertex)
- void UpdateLastPolygon(const QPoint& new_vertex) - соответствует методу UpdateLastVertex у Polygon.
- Геттер и сеттер для QPoint light_source.

Графический интерфейс приложения:

- Возможность переключения между двумя режимами работы приложения: **&lt;light&gt;** и **&lt;polygons&gt;**, например, при помощи QComboBox или QRadioButton.
- В обоих режимах доступна некая область для рисования. Между режимами содержимое области не меняется, но меняется способ взаимодействия пользователя с ней.
- Режим **&lt;light&gt;**:
  - Отрисовывается точка, которой управляет пользователь. Она может следовать за курсором, либо ей можно управлять при помощи клавиатуры (выберите более предпочтительный для вас способ). В будущем эта точка будет служить источником света.
- Режим **&lt;polygons&gt;**:
  - Первое нажатие ЛКМ: создается новый многоугольник.
  - Второе и последующее нажатия ЛКМ: добавляется новая вершина в текущий многоугольник.
  - Нажатие ПКМ: заканчиваем рисовать текущий многоугольник (после этого следующее нажатие ЛКМ трактуется как первое).
  - Многоугольники будут препятствиями для света, создающими тень.
- Данные должны храниться в объекте класса Controller.
- Для простоты будущих вычислений рекомендуется также добавить один большой фиктивный многоугольник с вершинами в краях области для рисования.

**2: Пускаем лучики (кастим рэи?) (7.5 баллов)**

Добавьте в Controller следующие методы:

- std::vector&lt;Ray&gt; CastRays() - создает из light_source в каждую вершину каждого многоугольника по три луча: луч, конец которого ровно в вершине, и две его копии, повернутые на ± 0.0001 радиан.
- void IntersectRays(std::vector&lt;Ray&gt;\* rays) - изменяет переданный вектор следующим образом: вызывает IntersectRay для каждого многоугольника с каждым лучом. Меняет конец луча, если расстояние от начала луча то результата выполнения IntersectRay меньше расстояния от начала луча до конца луча.
- void RemoveAdjacentRays(std::vector&lt;Ray&gt;\* rays) - удаляет из вектора лучи, расстояние между концами которых очень мало (конечно, один из таких лучей в векторе должен остаться, а вот все его близкие копии нужно удалить).
- Polygon CreateLightArea() - при помощи вышеназванных методов создает многоугольник, представляющий собой освещенную часть плоскости. _Подсказка: в каком-то из методов можно отсортировать лучи по величине угла._

Измените графический интерфейс, чтобы отрисовывать свет и тени.

**3: Добавляем полутени (5 баллов)**

- Теперь источников света должно быть несколько, и они должны быть расположены близко друг к другу (пользователь все еще управляет всеми источниками одновременно).
- Каждому из источников света теперь соответствует полупрозрачная освещенная плоскость. За счет наложения таких плоскостей друг на друга создастся эффект полутеней.

**\*: Дополнительные (необязательные) задания**

- Добавить третий режим работы приложения: **&lt;static-lights&gt;**. В нем при нажатии ЛКМ можно создавать дополнительные источники света - пользователь не может управлять их движением, но они также создают свет. Возможно, их свет будет иметь другой цвет или ограниченный радиус.
- Обрабатывайте коллизии между многоугольниками, а также между источниками света и многоугольниками, так что пользователь не может создать пересекающиеся многоугольники, или завести источник света внутрь многоугольника.

## Как сдавать лабораторную?

- **!!! Сделать новый branch отдельно от лабораторной работы №1 (от main в основном репозитории)**
- Внести изменения в labs/graphics (сами создаете структуру проекта по примеру, напр., отсюда: <https://github.com/pkasila/rules_qt6/tree/main/tests/uic_demo>)
- Проверить, что Ваш проект можно запустить с помощью просто bazel run //labs/raycaster (то есть имя конечной цели Вашего проекта должно быть basics, пример минимального BUILD-файла [тут](https://github.com/pkasila/bazel-qt-example/blob/main/demos/helloworld/BUILD))
- Закоммитить и закинуть всё в Ваш fork, кинуть PR в основной репозиторий (тот, который pkasila/bazel-qt-example)
